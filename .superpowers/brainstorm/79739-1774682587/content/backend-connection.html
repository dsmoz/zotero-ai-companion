How should the plugin talk to the MCP backend?

The existing MCP server runs as a separate process (Flask REST on :6500). The plugin needs to connect to it.

A

REST API only (port 6500)

Plugin calls the existing Flask endpoints directly. No changes to the MCP server — just consume what's already there. Fast to build.

Pros

  • Zero backend changes
  • Standard HTTP, easy to debug
  • Works today

Cons

  • Some features may need new endpoints
  • No streaming for chat (polling)
B

REST + Server-Sent Events for streaming

REST for queries/mutations. SSE stream for chat responses — AI tokens stream into the plugin in real time, like Claude Code's output. Requires adding SSE endpoints to Flask.

Pros

  • Chat feels live and responsive
  • Natural fit for LLM streaming

Cons

  • Small backend additions needed
C

WebSocket (persistent connection)

Full-duplex channel between plugin and backend. Enables real-time indexing progress, live graph updates as new items are added, streaming chat.

Pros

  • Real-time everything
  • Push notifications (new item indexed)

Cons

  • More complex to implement
  • Backend needs Flask-SocketIO
