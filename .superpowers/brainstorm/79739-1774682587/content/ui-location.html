Where should the plugin UI live?

Each approach has different integration depth and development complexity

A

Right Panel (Item Pane)

A new tab alongside "Info", "Notes", "Tags" — visible when you select an item. Shows AI chat, similar docs, metadata actions for the selected paper.

Pros

  • Context-aware to selected item
  • Standard Zotero 7 pattern
  • Low friction — always visible

Cons

  • Limited vertical space
  • Graph view would feel cramped
B

Floating / Detachable Panel

A dedicated window that can be docked or floated — houses the full companion: chat, graph, discovery, library health. Opened via toolbar button or menu.

Pros

  • Full space for graph viz
  • Multi-panel layout possible
  • Doesn't crowd main UI

Cons

  • Extra step to open
  • Context switch from reading
C

Hybrid: Item Pane tab + Full Window

Quick actions (chat, similar items) in the item pane tab. Heavy features (graph, discovery, bulk ops) open in a dedicated full window.

Pros

  • Right tool for each job
  • Low friction for common tasks
  • Graph/discovery get full space

Cons

  • More UI surface to design
  • Higher dev complexity
